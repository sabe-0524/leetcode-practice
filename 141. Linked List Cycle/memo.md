# LeetCode 141: Linked List Cycle 解法メモ

## Step 1
- **重要な発見**：LeetCode では `main` 関数を書く必要がない（これを知らずに大幅に時間を割かれた）
- 最初に思いついた解法：
  - `set` を使って訪問済みノードを記録し、重複が出たらサイクルありと判定
- 解答確認後の気づき：
  - `fast`（2ステップ進む）と `slow`（1ステップ進む）を用意し、同じノードに到達したらサイクルありと判定する方法が存在
  - この方法なら **メモリ使用量が減る**（O(1)）
